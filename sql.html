Introduction to SQL

SQL (Structured Query Language) is a standard language used to communicate with relational databases. It is used to store, retrieve, manage, and manipulate data.
Popular databases that use SQL include MySQL, PostgreSQL, Microsoft SQL Server, Oracle, and SQLite.

SQL Syntax

Basic syntax of an SQL query: 
SELECT column1, column2 FROM table_name WHERE condition;
SQL statements are not case-sensitive, but it's common to use uppercase for SQL keywords.

SELECT Statement

The SELECT statement is used to fetch data from a database table. 
SELECT * FROM Customers;
    SELECT name, email FROM Customers;
* selects all columns from the table.

SELECT DISTINCT

SELECT DISTINCT is used to return only distinct (different) values. 
SELECT DISTINCT city FROM Customers;
This query returns only unique cities from the Customers table.

WHERE Clause

The WHERE clause is used to filter records that fulfill a specified condition. 
SELECT * FROM Customers WHERE country = 'India';

ORDER BY

The ORDER BY clause is used to sort the result-set in ascending or descending order. 
SELECT * FROM Customers ORDER BY name ASC;
    SELECT * FROM Customers ORDER BY id DESC;
ASC is the default (ascending order), while DESC is used for descending order.

AND Operator

The AND operator is used to filter records based on more than one condition, all of which must be true. 
SELECT * FROM Customers WHERE country = 'India' AND city = 'Mumbai';

OR Operator

The OR operator returns records if at least one condition is true. 
SELECT * FROM Customers WHERE city = 'Mumbai' OR city = 'Delhi';

NOT Operator

The NOT operator is used to exclude rows that meet a specified condition. 
SELECT * FROM Customers WHERE NOT country = 'India';

INSERT INTO

The INSERT INTO statement is used to add new records to a table.

Syntax 1: Insert into all columns

INSERT INTO Customers VALUES (1, 'Aryan', 'Mumbai', 'India');

Syntax 2: Insert into specific columns

INSERT INTO Customers (name, city, country)
    VALUES ('Aryan', 'Mumbai', 'India');
It is recommended to specify column names for clarity and to avoid errors.

NULL Values

NULL in SQL represents missing or unknown data. It is different from an empty string or a zero value. SELECT * FROM Customers WHERE phone IS NOT NULL;
  • Use IS NULL and IS NOT NULL to filter records containing NULL values.

UPDATE Statement

The UPDATE statement is used to modify existing records in a table. 
UPDATE Customers
SET city = 'Pune'
WHERE name = 'Aryan';
‚ö†Ô∏è Always use a WHERE clause to avoid updating all rows unintentionally.

DELETE Statement

The DELETE statement is used to remove one or more records from a table. 
DELETE FROM Customers WHERE id = 5;
‚ö†Ô∏è Without a WHERE clause, all records will be deleted.

SELECT TOP

SELECT TOP is used to limit the number of rows returned. It is supported in SQL Server. 
SELECT TOP 3 * FROM Customers;
In MySQL, use:
SELECT * FROM Customers LIMIT 3;

Aggregate Functions

Aggregate functions perform calculations on a set of values and return a single result.

MIN() and MAX()

Return the smallest and largest values in a column 
SELECT MIN(age) FROM Customers;
SELECT MAX(age) FROM Customers;
.

COUNT()

Returns the number of rows that match a condition. 
SELECT COUNT(*) FROM Customers;
SELECT COUNT(city) FROM Customers WHERE country = 'India';
.

SUM()

Adds up all values in a column. 
SELECT SUM(balance) FROM Accounts;
.

AVG()

Calculates the average of a numeric column. 
SELECT AVG(salary) FROM Employees;

LIKE Operator

The LIKE operator is used in a WHERE clause to search for a specified pattern. 
SELECT * FROM Customers WHERE name LIKE 'A%';  -- Starts with A
SELECT * FROM Customers WHERE name LIKE '%n';  -- Ends with n

Wildcards

Wildcards are used with the LIKE operator to match patterns:
  • % ‚Äì represents zero or more characters
  • _ ‚Äì represents a single character
SELECT * FROM Products WHERE name LIKE '_a%';
Finds products where the second letter is "a".

IN Operator

The IN operator allows you to specify multiple values in a WHERE clause. 
SELECT * FROM Customers WHERE city IN ('Mumbai', 'Pune', 'Delhi');
This is the same as using multiple OR conditions.

BETWEEN Operator

BETWEEN selects values within a given range (inclusive). 
SELECT * FROM Orders WHERE order_date BETWEEN '2024-01-01' AND '2024-12-31';

SQL ALIASES

Aliases are used to rename a column or table for display or convenience. Use the AS keyword. SELECT o.id, c.name FROM Orders AS o JOIN Customers AS c ON o.customer_id = c.id;

SQL JOINS

Joins are used to combine rows from two or more tables, based on a related column.

INNER JOIN

Returns records that have matching values in both tables. 
SELECT Orders.id, Customers.name
FROM Orders
INNER JOIN Customers ON Orders.customer_id = Customers.id;

LEFT JOIN

Returns all records from the left table, and matched records from the right table. 
SELECT Customers.name, Orders.id
FROM Customers
LEFT JOIN Orders ON Customers.id = Orders.customer_id;

RIGHT JOIN

Returns all records from the right table, and matched records from the left table. 
SELECT Customers.name, Orders.id
FROM Customers
RIGHT JOIN Orders ON Customers.id = Orders.customer_id;

FULL JOIN

Returns all records when there is a match in either left or right table (not supported in MySQL directly). 
SELECT Customers.name, Orders.id
FROM Customers
FULL OUTER JOIN Orders ON Customers.id = Orders.customer_id;

SELF JOIN

A SELF JOIN is a regular join but the table is joined with itself. 
SELECT A.name AS Employee, B.name AS Manager
FROM Employees A, Employees B
WHERE A.manager_id = B.id;

UNION

The UNION operator is used to combine the result-set of two or more SELECT statements. Duplicate values are removed by default. 
SELECT city FROM Customers
UNION
SELECT city FROM Suppliers;

GROUP BY

GROUP BY groups rows that have the same values into summary rows, like "total sales per customer". 
SELECT customer_id, COUNT(*) AS order_count
FROM Orders
GROUP BY customer_id;

HAVING

HAVING is used to filter groups created by the GROUP BY clause. 
SELECT customer_id, COUNT(*) AS order_count
FROM Orders
GROUP BY customer_id
HAVING COUNT(*) > 5;

EXISTS

EXISTS checks whether a subquery returns any rows. 
SELECT name FROM Customers
WHERE EXISTS (
  SELECT * FROM Orders WHERE Customers.id = Orders.customer_id
);

ANY and ALL

ANY returns true if any of the subquery values meet the condition. ALL returns true if all values meet the condition. 
SELECT name FROM Products
WHERE price > ANY (SELECT price FROM Products WHERE category = 'Electronics');

SELECT name FROM Products
WHERE price > ALL (SELECT price FROM Products WHERE category = 'Accessories');

SELECT INTO

SELECT INTO copies data from one table into a new table. 
SELECT * INTO CustomersBackup
FROM Customers;

INSERT INTO SELECT

Used to copy data from one table to another existing table. 
INSERT INTO CustomersArchive (id, name, city)
    SELECT id, name, city FROM Customers
    WHERE country = 'India';

CASE Statement

CASE is SQL‚Äôs way of handling conditional logic 
SELECT name, 
CASE 
  WHEN marks >= 75 THEN 'Distinction'
  WHEN marks >= 60 THEN 'First Class'
  ELSE 'Pass'
END AS Grade
FROM Students;

NULL Functions

Functions like ISNULL(), COALESCE(), and IFNULL() handle NULL values. 
SELECT name, ISNULL(phone, 'Not Available') FROM Customers;
SELECT name, COALESCE(email, alt_email, 'No Email') FROM Users;

Stored Procedures

A Stored Procedure is a set of SQL statements that can be saved and reused. 
CREATE PROCEDURE GetCustomers
AS
BEGIN
  SELECT * FROM Customers;
END;

-- Execute the procedure
EXEC GetCustomers;

SQL Operators

Operators are used to perform operations on data:
  • Arithmetic: +, -, *, /, %
  • Comparison: =, <>, >, >=, <, <=
  • Logical: AND, OR, NOT
SELECT * FROM Products WHERE price > 100 AND stock <> 0;

Create Database

Creates a new SQL database. 
CREATE DATABASE SchoolDB;

Drop Database

Deletes an existing database. Be careful! 
DROP DATABASE SchoolDB;

Backup Database

Creates a backup of an existing database (mainly used in SQL Server). 
BACKUP DATABASE SchoolDB
TO DISK = 'D:\backups\SchoolDB.bak';

Create Table

Creates a new table in the database. 
CREATE TABLE Students (
  id INT PRIMARY KEY,
  name VARCHAR(100),
  age INT,
  email VARCHAR(100)
);

Drop Table

Deletes an existing table and all its data. 
DROP TABLE Students;

Alter Table

Modify an existing table structure (add, modify, drop columns). 
-- Add a new column
ALTER TABLE Students ADD phone VARCHAR(15);

-- Modify a column datatype
ALTER TABLE Students MODIFY age SMALLINT;

-- Drop a column
ALTER TABLE Students DROP COLUMN phone;

Constraints

Constraints define rules for data in a table.

NOT NULL

Ensures that a column cannot have NULL values. 
CREATE TABLE Employees (
  id INT PRIMARY KEY,
  name VARCHAR(100) NOT NULL
);

UNIQUE

Ensures all values in a column are unique. 
CREATE TABLE Employees (
  email VARCHAR(100) UNIQUE
);

Primary Key

Uniquely identifies each row in a table. 
CREATE TABLE Departments (
  dept_id INT PRIMARY KEY,
  dept_name VARCHAR(100)
);

Foreign Key

Ensures referential integrity between two tables. 
CREATE TABLE Orders (
  order_id INT PRIMARY KEY,
  customer_id INT,
  FOREIGN KEY (customer_id) REFERENCES Customers(id)
);

CHECK

Ensures that all values in a column satisfy a condition. 
CREATE TABLE Employees (
  salary DECIMAL(10,2),
  CHECK (salary >= 0)
);

DEFAULT

Provides a default value for a column when none is specified. 
CREATE TABLE Products (
  id INT PRIMARY KEY,
  price DECIMAL(10,2) DEFAULT 0.00
);

Index

Improves the speed of data retrieval operations. 
CREATE INDEX idx_name ON Employees(name);

Auto Increment

Automatically generates a unique number for new rows (syntax may vary). 
CREATE TABLE Users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  username VARCHAR(50)
);

Dates and Date Functions

SQL offers many functions to work with dates and times.
  • GETDATE() - Returns current date and time (SQL Server)
  • CURDATE() - Returns current date (MySQL)
  • DATEADD() - Adds a specified time interval to a date
  • DATEDIFF() - Calculates difference between two dates
  • DATE_FORMAT() - Formats date values (MySQL)
SELECT GETDATE();  -- SQL Server
SELECT CURDATE();   -- MySQL

SELECT DATEADD(day, 7, GETDATE());  -- Adds 7 days

SELECT DATEDIFF(day, '2024-01-01', '2024-06-01');  -- Days between dates

SELECT DATE_FORMAT(NOW(), '%Y-%m-%d');  -- Formats date

Views

A View is a virtual table based on the result set of an SQL statement. 
CREATE VIEW ActiveCustomers AS
SELECT * FROM Customers WHERE status = 'Active';

SELECT * FROM ActiveCustomers;

SQL Injection

SQL Injection is a security vulnerability that allows attackers to interfere with queries.
Always use parameterized queries or prepared statements to prevent it.

SQL Hosting

Databases can be hosted on local servers or cloud providers like AWS RDS, Azure SQL Database, Google Cloud SQL.
Choosing a host depends on scalability, security, cost, and performance needs.

Data Types

SQL supports various data types including:
  • Numeric: INT, FLOAT, DECIMAL
  • String: CHAR, VARCHAR, TEXT
  • Date/Time: DATE, TIME, DATETIME, TIMESTAMP
  • Boolean: BIT, BOOLEAN
